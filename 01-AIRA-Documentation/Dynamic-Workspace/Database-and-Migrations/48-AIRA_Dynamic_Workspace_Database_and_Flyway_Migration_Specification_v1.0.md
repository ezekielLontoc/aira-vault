### 1.3 1. Executive Summary

This specification defines the authoritative PostgreSQL schema and Flyway migration strategy for the AIRA Dynamic User Workspace and Composable Experience Framework. The database stores workspace templates, screen templates, component catalog entries, rendering policies, experience blocks, experience packs, capability bindings, AI capabilities, workflow bindings, evidence profiles, layout definitions, and user personalization overlays.

PostgreSQL is the authoritative source. Redis/Valkey is a derivative cache only.

### 1.9 7. Seed Data Rules

Seed data must be:

- Git-controlled;

- loaded through Flyway or approved configuration seeder;

- validated before insert;

- assigned owner and evidence reference;

- protected from direct production edits;

- versioned and reversible.

### 1.10 8. Rollback and Forward-Fix Rules

- Schema rollback in production must prefer forward-fix migration.

- Configuration rollback must activate previous approved version.

- Layout rollback must record actor, reason, old version, new version, and evidence reference.

- No direct manual DB edits are allowed for production correction.
